# DINEHUB: Restaurant-Management-with-UI-
A full-stack restaurant management system built with a MySQL relational database at its core, a Python Flask REST API backend, and a vanilla HTML/CSS/JavaScript frontend. The system automates and manages all core restaurant operations through intelligent AI agents and database-level automation.

**Overview:**
DineHub simulates a real-world restaurant environment where customers can browse a live menu, place dine-in or delivery orders, make table reservations, and join a walk-in queue, all connected to a normalized relational database that handles business logic automatically through triggers, views, and stored procedures.
The project was built as a Database Systems course project to demonstrate the practical application of database design principles, including normalization, referential integrity, indexing, triggers, and views in a real-world scenario.

**Key Features:**
Relational database design and normalization up to 3NF
Entity-Relationship modeling with composite and surrogate keys
Database triggers for automated business logic
SQL views for reporting and dashboard queries
Indexing strategy for query performance
RESTful API design with Flask blueprints
JWT-based stateless authentication
AI agent design patterns for resource allocation
Frontend-backend-database three-tier architecture
